FLIXLINK Documentation

WHAT IS FLEXLINK?

FlexLink provides users with information about different movies, directors, and genres. The server-side of the FlixLink application will ensure they have access to this information, that their requests can be processed, and that all necessary data can be stored.

Users will be able to create an account with which to enter, maintain, and update their personal information, as well as create a customized list of their favorite movies.


FlexLink API Documentation

This documentation describes the endpoints and their functionality for managing users and their favorite movies using the Flexlink API.

Create (POST) Functions

Add New User

  • URL: /users
  • Method: POST
  • Description: Adds a new user to the system.
  • Request Body:
    • Creates the following data categories for the user registry:
      • UserName [required]
      • Password [required]
      • Email [required]
      • Birthday
      • Favorite Movies
      • User ID [automatically generated]


      SAMPLE:

      {
      "UserName": "sampleusername",
      "Password": "samplepassword1234!",
      "Email": "sample@gmail.com",
      "Birthday": "01/01/2002"
      }

  • Responses:
    • 201 (Created): Displays the new user data to show it was successful.
    • 500 (Bad Request): Error: ValidationError: Email: Path '[required path name]' is required.

Add Movie to User Favorites

  • URL: /users/:UserName/movies/:MovieID
  • Method: POST
  • Description: Adds a movie to a user's list of favorite movies.
  • URL Parameters:
    • UserName (string) - Username.
    • MovieID (string) - The object ID of the movie title to be added to favorites.
  • Responses:
    • 200 (OK): Returns the updated user data reflecting the added favorite movie.
    • 500 (Bad Request): Movie fails to be added.


Read (GET) Functions

Get All Movie Data

  • URL: /movies
  • Method: GET
  • Description: Returns a list of all movies in the database.
  • Response Body:

    • SAMPLE:

      {
      "Genre": "ObjectID of assigned Genre",
      "Director": "ObjectID of assigned Director",
      "_id": "[ObjectID of movie]",
      "Title": "Title of Movie",
      "Description": "Movie Description"
      "Image URL": "Location of the representative image",
      "Featured": "true/false"
      }

Get Movie Data on a Specified Movie

  • URL: /movies/:Title
  • Method: GET
  • Description: Returns all data on the specified name of the the movie.
  • URL Parameters:
    • title (string) - Title of the movie.
  • Response Body:

    • SAMPLE:

      {
      "Genre": "ObjectID of assigned Genre",
      "Director": "ObjectID of assigned Director",
      "_id": "[ObjectID of movie]",
      "Title": "Title of Movie",
      "Description": "Movie Description"
      "Image URL": "Location of the representative image",
      "Featured": "true/false"
      }

Get All Genre Data

  • URL: /genres
  • Method: GET
  • Description: Returns a list of all genres in the database.
  • Response Body:

    • SAMPLE:

      {
      "_id": "ObjectID of genre",
      "Name": "Title of Genre",
      "Description": "Genre Description"
      }

Get Data on a Specified Movie Genre

  • URL: /genres/:genreName
  • Method: GET
  • Description: Returns all data on the specified genre.
  • URL Parameters:
    • genreName (string) - Name of the genre.
  • Response Body:
    • {
      "_id": "ObjectID of the genre",
      "Name": "Genre Name",
      "Description": "Genre Description"
      }

Get All Director Data

  • URL: /directors
  • Method: GET
  • Description: Returns a list of all directors in the database.
  • Response Body:

    • SAMPLE:

      {

      "Name": "Name of the director",
      "Bio": "Biographical Description"
      "Birth": "Director date of birth",
      "Death": "Director date of death (if applicable)",
      }

Get Data on a Specified Movie Director

  • URL: /directors/:directorName
  • Method: GET
  • Description: Returns all data on the specified director.
  • URL Parameters:
    • directorName (string) - Name of the director.
  • Response Body:
    • {
      "_id": "ObjectID of the director",
      "Name": "Name of the director",
      "Bio": "Biographical Description"

      "Death": "Director date of death (if applicable)",
      }

Update (PUT) Functions

Update User Info (Username)

  • URL: /users/:username
  • Method: PUT
  • Description: Updates the data of an existing user.
  • URL Parameters:
    • UserName (string) - Current UserName.
  • Request Body:
    • Updated user object data:

      {
      "UserName": "Updated Name",
      "Password": "Updated Password",
      "Email": "Updated Email",
      "Birthday": "Updated birthday [01/01/2001]"
      }

  • Responses:
    • Send updated user data:
      • {
        "id": "ObjectID of user",
        "UserName": "Updated username",
        "Password": "Updated password",
        "Email": "Updated email",
        "Birthday": "Updated birthdate",
        "FavoriteMovies": "Current favorites",
        }

    • 500 (Bad Request): Returns a server error.

Delete (DELETE) Functions

Delete User Account

  • URL: /users/:UserName
  • Method: DELETE
  • Description: Deletes a user from the database.
  • URL Parameters:
    • UserName (string) - Username.
  • Responses:
    • 200 (OK): [Username] was deleted.
    • 400 (Bad Request): [Username] was not found.
    • 500 (Bad Request): Returns a server error.

Delete Movie from Favorites

  • URL: /users/:UserName/movies/:MovieID
  • Method: DELETE
  • Description: Removes a movie from a user's list of favorite movies.
  • URL Parameters:
    • UserName (string) - Username.
    • MovieID (string) - ObjectID of Movie to be removed from favorites.
  • Responses:
    • Returns updated user data.
    • 500 (Bad Request): Returns a server error.
